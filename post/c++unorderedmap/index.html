什么是unorderedmap？

  • 一个关联容器，包含具有唯一键的键值对

使用unorderdmap有什么好处？

  • 元素的搜索、插入和删除时间复杂度O(1)

原理是什么？

  • 在内部，元素不按任何特定顺序排序，而是组织到存储桶中
  • 元素放入哪个存储桶完全取决于其键的哈希值，具有相同哈希码的键出现在同一个存储桶中

如何使用？

初始化

std::unordered_map<std::string, std::string> u =
    {
        {"RED", "#FF0000"},
        {"GREEN", "#00FF00"},
        {"BLUE", "#0000FF"}
    };

相关函数

  • operator= 为容器赋值
  • begin/cbegin 返回指向开头的iterator
  • end/cend 返回指向结尾的迭代器
  • empty 检查容器是否为空
  • size 返回元素的数目
  • clear 清空容器
  • insert 插入元素或节点(C++17以后)
  • insert_range(C++23) 插入一系列元素
  • erase 擦除元素
  • extract(C++17) 从其他容器中提取节点
  • merge(C++17) 从其他容器中接入节点
  • operator[] 获取特定元素
  • count 返回能匹配到特定key的元素数目
  • find 根据key寻找元素
    • 返回迭代器
  • bucket_count 返回桶的数目
  • max_bucket_count 返回最大的桶的数目
  • bucket_size 返回指定桶的大小
  • bucket 返回指定的桶
  • load_factor 返回每个桶拥有的元素的数目的平均数
  • hash_function 返回对key进行操作的哈希函数
  • operator==(C++11)
  • operator!=(C++11)

特别的用法

  • operator[]
unordered_map hashtable;
hashtable[1] = 2;
// 相当于添加(1, 2)这个元素
