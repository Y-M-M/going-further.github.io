第一章 Rust入门

1.1 Rust简介

  • 可靠+高效
  • 可用来替换C/C++，很多常见的bug在编译时就可以被消灭
  • 适用于以下场景
    • 需要运行时的速度
    • 需要内存安全
    • 更好的利用多处理器
  • 与其他语言比较
    • C/C++ 性能非常好，但类型系统和内存都不太安全
    • Java/C# 拥有GC，能保证内存安全，也有很多优秀特性，但性能不行
    • Rust
      • 安全
      • 无需GC
      • 易于维护、调试，代码安全高效
  • 特别擅长的领域
    • 高性能Web Service
    • WebAssembly
    • 命令行工具
    • 网络编程
    • 嵌入式设备
    • 系统编程
  • Rust 与 FireFox
  • Rust的用户和案例
    • Google
    • Amazon
    • System76
    • 蚂蚁金服
    • 斯坦福和密歇根大学
    • 微软
  • Rust的优点
    • 性能
    • 安全性
    • 无所畏惧的并发
  • 缺点
    • 难学
  • 注意
    • Rust有很多独有的概念，和现在大多数主流语言都不同

1.4 Cargo

什么是Cargo?

  • Cargo是Rust的构建系统和包管理工具
    • 构建代码、下载依赖的库、构建这些库…
  • 安装Rust的时候会安装Cargo
    • cargo –version

使用Cargo创建项目

  • cargo new hello_cargo

    • 创建项目
      • .toml
      • src目录
        • main.rs
      • 初始化git仓库
        • .gitgnore

  • Cargo.toml

    • TOML格式，是Cargo的配置格式
    • [package]是一个区域标题，表示下方内容使用来配置包的
      • name 项目名
      • version 项目版本
      • edition 使用Rust版本
    • dependencies 项目依赖项
    • 在Rust里面，代码的包称作create

  • src/main.rs

    • 源代码都放在src目录下
[package]
name = "hello_cargo"
version = "0.1.0"
edition = "2021"

[dependencies]

使用Cargo构建项目

  • cargo build
    • 创建可执行文件 target/debug/可执行文件名
  • 第一次运行cargo build会在顶层目录生成cargo.lock文件
    • 该文件负责追踪项目依赖的精确版本
    • 不需要手动修改该文件

构建+运行cargo项目

  • cargo run
    • 编译代码+执行结果
    • 如果之前编译成功过，并且源代码没有改变，那么就会直接运行二进制文件

检查

  • cargo check
    • 检查代码，确保能通过编译，但是不产生任何可执行文件
    • 要比cargo build快得多
      • 编写代码的时候可以连续反复的使用cargo check检查代码，提高效率

为发布构建

  • cargo build –release
    • 编译时会进行优化
      • 代码会运行的更快，但是编译时间更长
    • 会在target/release而不是target/debug生成可执行文件

!尽量使用Cargo

猜数游戏

2.1 一次猜测

  • 目标
    • 生成一个1到100间的随机数
    • 提示玩家输入一个猜测
    • 猜完之后，程序会提示猜测是太小了还是太大了
    • 如果猜测正确，那么打印出一个庆祝信息，程序退出
use std::io; // prelude 获得用户输入/打印输出

fn main() {
    println!("猜数!"); // 宏，输出
    println!("猜测一个数");

    // let mut foo = 1;
    // let bar = foo; // immutable 默认不可变
    // foo = 2;

    let mut guess = String::new(); // 返回字符串的标准实例(utf-8编码),可以按照需求扩展大小
    // ::表明new()是String的一个关联函数，针对类型本身来实现，new创建空白字符串

    io::stdin().read_line(&mut guess).expect("无法读取行");
    // 将用户输入赋值给字符串guess，guess必须是可变来行的，&表示是字符串的引用
    // 引用在Rust里默认不可变 mut让引用变成可变
    // readline 返回io::Result<usize>
    // io::Result 枚举类型 Ok操作成功, Err操作失败，附带失败原因
    // expect，Result上定义的方法，假如io::Result返回的是err，expect将中断这个信息，并显示传入的字符串信息；如果返回Ok，将提取Ok中附加的值，返回给用户

    println!("你猜测的数是: {}", guess);
    // 花括号：占位符，会被替换成后面的变量的值
}
